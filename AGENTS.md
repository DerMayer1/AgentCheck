# AgentCheck contributor instructions

## Product boundary

AgentCheck is a deterministic, local-only CLI. `agentcheck scan` must never execute target-repository code, write inside the target, follow symlinks outside it, or make network requests.

## Development commands

- Install dependencies with `npm install`.
- Run the canonical validation sequence with `npm run verify`.
- Run the CLI during development with `npm run dev -- scan [path]`.
- Build the distributable CLI with `npm run build`.

## Change rules

- Keep domain, detector, scoring, and reporter responsibilities separate.
- Repository rules must consume normalized facts rather than traverse files directly.
- Treat rule IDs, exit codes, score semantics, and JSON output as public API.
- Add fixtures for passing, failing, skipped, malformed, and boundary behavior before shipping a rule.
- Do not edit `dist/`; it is generated by the build.
- Do not add runtime dependencies when a small, auditable implementation is practical.

## Completion

A change is complete only when `npm run verify` passes. CLI behavior changes also require a smoke test against the built `dist/cli/main.js` entry point.
